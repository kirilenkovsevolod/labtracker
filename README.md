# LabTracker

**АРМ преподавателя для управления лабораторными работами**

Система учёта сдачи лабораторных работ с Telegram-ботом для студентов и Qt-интерфейсом для преподавателя. Студент получает случайный контрольный вопрос, отвечает в боте — нейросеть проверяет ответ и засчитывает сдачу. Учебный проект по дисциплине «Информатика».

---

## Возможности

- **Авторизация** студентов через Telegram-бот (логин/пароль, привязка Telegram ID).
- **Каталог лабораторных** — преподаватель назначает лабы группам через Qt-интерфейс, выбирая из готовых шаблонов. К каждой лабе прикреплены контрольные вопросы с эталонными ответами.
- **AI-проверка ответов** — при сдаче студент получает случайный вопрос по теме. Нейросеть сравнивает ответ с эталоном и выносит вердикт YES / NO.
- **Файлы лабораторных** — преподаватель прикрепляет PDF/DOCX к лабораторной; студент скачивает прямо из бота.
- **Дедлайны и напоминания** — бот автоматически уведомляет студентов за 2 и за 1 день до срока сдачи.
- **Панель преподавателя (Qt)** — просмотр студентов, лабораторных и сдач в табличном виде; добавление и удаление записей; кнопка «Обновить».
- **Статусы сдач** — ✅ сдано / ❌ не сдано; отображается в боте и в Qt-панели.

## Архитектура

```
LabTracker/
├── main.py          FastAPI-сервер, все эндпоинты, модели БД
├── bot.py           Telegram-бот (авторизация, сдача, напоминания)
├── uploads/         Файлы лабораторных (PDF, DOCX)
├── .env             Токены и ключи API (не коммитится)
├── test.db          SQLite-база данных
└── qt_gui/
    ├── main.cpp
    ├── mainwindow.h / .cpp     Главное окно (3 вкладки)
    ├── addlabdialog.h / .cpp   Диалог назначения лабораторной
    ├── addstudentdialog.h / .cpp
    └── labtracker.pro
```

Взаимодействие компонентов:

```
Telegram Bot (bot.py)
        ↕  HTTP
FastAPI Server (main.py)  ←→  SQLite (test.db)
        ↕  HTTP
Qt GUI (C++)
```

Бот и Qt-интерфейс не общаются напрямую — оба работают только через HTTP API сервера.

## Установка и запуск

### Требования

- Python 3.11+
- Qt 6 + Qt Creator
- Аккаунт Groq (бесплатный ключ API)
- Telegram Bot Token ([@BotFather](https://t.me/BotFather))

### Установка зависимостей

```bash
pip install fastapi uvicorn sqlalchemy python-telegram-bot openai python-dotenv python-multipart "python-telegram-bot[job-queue]"
```

### Настройка

Создать файл `.env` в корне проекта:

```
BOT_TOKEN=ваш_токен_бота
GROQ_API_KEY=ваш_ключ_groq
GROQ_MODEL=llama-3.1-8b-instant
```

### Запуск сервера

```bash
uvicorn main:app --reload
```

### Запуск бота

```bash
python bot.py
```

### Сборка Qt GUI

```bash
cd qt_gui
qmake labtracker.pro
make        # или mingw32-make на Windows
```

Или открыть `labtracker.pro` в Qt Creator и нажать ▶️.

## Сценарий использования

1. Преподаватель добавляет студентов через Qt-панель (имя, логин, пароль, группа).
2. Преподаватель назначает лабораторные группам: выбирает из шаблонов, задаёт дедлайн, прикрепляет файл.
3. Студент пишет `/start` в боте, вводит логин и пароль.
4. В меню «📚 Лабораторные» видит список своих лаб со статусами и дедлайнами.
5. Нажимает на лабу → видит описание и кнопку скачать файл.
6. Нажимает «Сдать» → бот задаёт случайный контрольный вопрос.
7. Студент пишет ответ → нейросеть проверяет → лаба засчитывается или нет.
8. За 2 и за 1 день до дедлайна бот напоминает о несданных лабах.
9. Преподаватель видит все сдачи в Qt-панели в реальном времени.
